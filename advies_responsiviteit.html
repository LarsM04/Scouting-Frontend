Zelfevaluatie Teamleider Scouting
Klant: Scouting Nederland
Datum: 25 juni 2026
Project: Zelfevaluatie-tool voor teamleiders
Auteur: Lars Mudde

1. Inleiding

Dit verslag beschrijft hoe de website "Zelfevaluatie Teamleider Scouting" responsive is gemaakt. Responsive betekent dat de website er goed uitziet en goed werkt op elk schermformaat: van een kleine telefoon tot een groot computerscherm. Hieronder leg ik uit welke keuzes ik heb gemaakt en waarom.

2. Gekozen aanpak: Mobile-First met Bootstrap 5

Voor dit project heb ik gekozen voor een mobile-first aanpak. Dit betekent dat ik eerst heb ontworpen voor kleine schermen (telefoons) en daarna heb aangepast voor grotere schermen (tablets en desktops).

Waarom mobile-first?
De meeste teamleiders bij Scouting zullen deze tool waarschijnlijk op hun telefoon gebruiken, bijvoorbeeld tijdens een bijeenkomst of op kamp. Door eerst voor mobiel te ontwerpen, weet je zeker dat de belangrijkste functies altijd goed werken op het kleinste scherm.

Als basis gebruik ik Bootstrap 5, een CSS-framework dat standaard al responsive is. Bootstrap werkt met een grid-systeem en biedt kant-en-klare klassen (zoals container, d-flex, d-grid) die ervoor zorgen dat elementen zich automatisch aanpassen aan het scherm.

3. Breakpoints en media queries

Wat is een breakpoint?

Een breakpoint is een schermgrootte waarbij het ontwerp verandert. Je stelt dit in met een media query in CSS. Zodra het scherm kleiner of groter wordt dan een bepaald punt, worden andere stijlregels actief.

Gebruikte breakpoints

Breakpoint Schermgrootte Doelgroep
max-width: 575.98px Kleiner dan 576px Telefoons (staand)
Bootstrap standaard (576px–768px) Kleine tablets Via Bootstrap-klassen
Bootstrap standaard (768px+) Tablets en desktops Via Bootstrap-klassen

Het hoofdbreakpoint dat ik zelf in CSS heb ingesteld is 575.98px. Dit is de grens voor kleine telefoons. Bootstrap regelt de rest via zijn eigen ingebouwde breakpoints.

Waarom deze keuzes?

  • 575.98px is de standaard grens die Bootstrap hanteert voor extra-kleine schermen. Dit past goed bij de schermgroottes van de meeste moderne telefoons (iPhone, Samsung, etc.).
  • De website hoeft niet heel veel verschillende layouts te hebben, omdat het in de basis een formulier is dat je van boven naar beneden doorloopt. Daarom is één eigen breakpoint voldoende.
  • Bootstrap handelt de tussenliggende formaten (tablets, kleine laptops) automatisch af met zijn grid-systeem.

4. Aanpassingen voor mobiele telefoons

Hieronder beschrijf ik per onderdeel wat ik heb gedaan om de website goed werkbaar te houden op mobiel:

4.1 Touch-targets (knoppen groot genoeg voor vingers)

Alle knoppen hebben een minimale hoogte van 44px tot 48px. Dit is de richtlijn van Apple en Google voor comfortabele touch-bediening. Specifiek:

  • Schaalbalkknoppen (1 t/m 5): minimaal 46px hoog op mobiel
  • Navigatieknoppen: minimaal 44px hoog
  • CTA-knoppen ("Volgende vraag", "Starten"): minimaal 48px hoog

4.2 Leesbare tekst

  • Op kleine schermen wordt de lettergrootte van de schaalknoppen iets verkleind (van 1.15rem naar 1.05rem) zodat alles goed past zonder dat de knoppen te krap worden.
  • Kopjes van taken (hoofdtaak-headers) worden ook iets kleiner (1.05rem in plaats van 1.08rem).

4.3 Genoeg ruimte aan de zijkanten

Op mobiel krijgt de container 20px padding links en rechts. Dit voorkomt dat tekst en knoppen te dicht tegen de rand van het scherm zitten, wat op telefoons al snel onleesbaar wordt.

4.4 Kaarten (module-cards)

De padding binnen de kaarten wordt op kleine schermen iets verkleind (van 20px 24px naar 18px 20px). Zo blijft er genoeg ruimte, maar gaat er niet te veel schermruimte verloren.

4.5 Logo

Het Scouting-logo op het startscherm wordt op telefoons iets kleiner weergegeven (van 100px naar 85px hoog). Dit zorgt ervoor dat de keuze-knoppen sneller in beeld zijn zonder dat de gebruiker hoeft te scrollen.

4.6 Flexbox en Grid

Door gebruik te maken van CSS Flexbox (d-flex, d-grid, gap-2) passen elementen zich automatisch aan. De schaalknoppen verdelen zich gelijkmatig over de beschikbare breedte, ongeacht het schermformaat.

4.7 Viewport-eenheden

De hoogte van het selectiescherm is ingesteld op 100dvh (dynamic viewport height). Dit zorgt ervoor dat het scherm altijd precies de volledige hoogte van het scherm vult, ook op telefoons waar de adresbalk soms verschuift.

5. Hoe ik heb getest

De website is getest op meerdere manieren:

  • Chrome DevTools — Hiermee heb ik verschillende apparaten gesimuleerd (iPhone SE, Galaxy S20, iPad, etc.)
  • Firefox en Edge — Om te controleren of de website er in verschillende browsers goed uitziet
  • Verschillende schermgroottes — 320px, 375px, 414px, 768px, 1024px en 1920px

6. Samenvatting

Keuze Reden
Mobile-first aanpak Gebruikers vullen de tool vooral op hun telefoon in
Bootstrap 5 grid-systeem Betrouwbare responsiviteit zonder veel extra code
Breakpoint op 575.98px Standaard voor kleine telefoons, past bij moderne apparaten
Grote knoppen (44–48px) Makkelijk te bedienen met je vinger
Flexbox layout Elementen passen zich automatisch aan
Aanpasbare padding en lettertypes Genoeg ruimte op klein scherm zonder verspilling
De website is zo gebouwd dat hij op elk apparaat goed werkt. De belangrijkste focus lag op telefoons, omdat teamleiders de tool het vaakst "in het veld" zullen gebruiken.
